# Cloud SQL まとめ

## 概要
- 対象: マネージドRDB（MySQL / PostgreSQL / SQL Server）
- 主用途: 高互換のRDBが必要な一般的な業務システム、小〜中規模、既存アプリ移行

## 利点
- 高互換性: 既存DBエンジン準拠で移行容易
- 運用省力: バックアップ、パッチ、フェイルオーバー、監視がマネージド
- 構成容易: シンプルなHA/レプリカ構成が短時間で作成可

## 欠点
- スケール限界: 書き込みは単一プライマリ依存、主に縦スケール
- グローバル強整合: クロスリージョン同期書き込みは不可
- 接続制約: 接続数/スロット、メンテナンスウィンドウの影響

## 価格の利点
- 導入しやすい: 小〜中規模でコスト/運用のバランスが良い
- 運用コスト削減: 自前運用に比べて人件費/リスクを低減

## 価格の欠点
- スケールで高額化: vCPU/メモリ、IOPS、ストレージ、レプリカで積み上がる
- 常時課金: アイドルでもインスタンス費用が発生

## 選定の目安
- 向く: 既存RDBのクラウド移行、互換性/シンプルな運用が最優先
- 代替: 高性能/PG互換→ AlloyDB、グローバル強整合→ Cloud Spanner

## コスト最適化 Tips
- 接続プール/コネクタ活用でコスト効率（アプリ側プール、Serverless VPC Connector等）
- 読み取りはレプリカへ、適切なマシンタイプ/ストレージIOPS選定
- バックアップ/ログの保持期間見直し、メンテ時間帯の調整
