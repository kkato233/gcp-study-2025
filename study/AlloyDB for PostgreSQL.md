# AlloyDB for PostgreSQL まとめ

## 概要
- 対象: PostgreSQL 互換のマネージド高性能RDB
- 主用途: 既存PG資産の高性能化、分析とトランザクションのハイブリッド、リージョン内HA

## 利点
- 互換性: PostgreSQL互換でアプリ改修が小さく移行しやすい
- 性能/可用性: 高速ストレージ層と読み取りプール、リージョン内HA、オンライン運用機能
- 運用性: バックアップ/パッチ/監視のマネージド化、シームレスな拡張

## 欠点
- グローバル強整合: マルチリージョンの同期書き込みは不可（主にリージョン内）
- スケール: 書き込みは単一プライマリ前提（スケールは縦＋分割設計）
- 設計負荷: レプリカ/ストレージ/メンテ/拡張方針の検討が必要

## 価格の利点
- 性能/コスパ: vCPUあたり性能が高く、同等性能を他手段で出すより有利なことが多い
- 移行コスト: PG互換により再開発コストを抑制

## 価格の欠点
- 常時課金: 24/7でリソース課金、アイドル時も費用発生
- 追加費用: レプリカ/ストレージ/ネットワークで積み上がる

## 選定の目安
- 向く: PG互換必須、高性能/HAが必要、グローバル書き込みは不要
- 代替: グローバル強整合/水平スケールが必須→ Cloud Spanner、最小変更で可→ Cloud SQL

## コスト最適化 Tips
- 読み取りプールの活用でプライマリ負荷を低減
- ワークロードに合わせたマシンサイズ/ストレージ階層
- バックアップ/ログ/保持期間の最適化、定期メンテの時間帯調整
