オンライン請求サービス「NovaPayments」は、請求書データをオンプレミスから Cloud
Storage に移行します。以下の要件をすべて満たしつつ、運用コストを最小限に抑える
構成を選ぶにはどうすればよいでしょうかぅ

・データは最低 5 年間保持する

・最新5 つのバージョンを管理する

・1 年以上経過したファイルは低コストストレージに自動移行する

a. Cloud Storage にオブジェクトメタデータを書き込む Function を作成し、バージョン数
と経過日数に応じて削除処理を実施する

b. Cloud Storage バケットに保持ポリシーを設定し、Cloud Scheduler を使って古ファイル
を手動で削除する

c. Cloud Storage でオブジェクトバージョン管理を有効にし、ライフサイクルルールで
バージョン数の制限とストレージクラスの自動移行を行い、保持ポリシーで5 年間削除
を禁止する

d. Cloud Storage でバージョンを手動で管理し、1 年経過後に低頻度アクセスストレージ
に手動で移動する
